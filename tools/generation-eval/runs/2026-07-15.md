6 assessed as true docs gaps at review.

forbidden), built the Profile-settings form end to end using correct components and **zero
invented props** — even Tag's `subtle` variant was pulled from `manifest.json` rather than

and `guidance.json` correctly steered it to Save→`accent` / Cancel→`neutral`. Where the docs ran

1. form-field composition — no Field component or recipe, so label/help/error-text wiring was
   undocumented.
2. stack/flex container pairing — gap utilities are documented without the flex/grid container
   they require, so no stack recipe existed to follow.

   leaving nested-mount theming ambiguous.
4. native prop pass-through — the manifest lists only component-specific props (e.g. 4 for
   Input), so passing `value`/`onChange`/`type` was a leap of faith.
5. "secondary"→neutral mapping — the variant vocabulary has no primary/secondary, so Cancel's
   "secondary" role had to be inferred onto `neutral`. **Assessed at review: covered by
   guidance.json** ("neutral = Secondary actions" already steers this mapping) — a vocabulary
   translation, not a docs gap. Not counted in the assessed docs-gap total.
6. Tag variant selection for the plan badge — which variant reads as a badge was unstated.
   **Assessed at review: partially covered by guidance.json** (accent-subtle intent points at
   this use) — not counted in the assessed docs-gap total.
7. size selection — 24/32/40/48 exist with no guidance on which to pick, so sizing was
   unguided.
8. semantic token roster — the docs abbreviate to globs (`--ds-fg-*`), forcing a grep of
   `dark.css` to find `--ds-fg-danger`.

Items 5 and 6 are excluded from the docs-gap total per the assessment notes above. This run is
the source of the "Compositions" recipes later added to `packages/react/llms.txt`.
